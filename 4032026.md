### 🛠️ Arquitectura de Sincronización del Historial (Aprobada)
Se ha definido una estrategia de **"Actualización Silenciosa y Automática"** coordinada con las GitHub Actions:

1. **Flujo de Datos**:
   - El script `main.py`, que ya accede al repo privado `lima-airport-data`, consolidará el historial.
   - Durante el scraper, Python leerá el archivo `alertas_enviadas.txt` del repo privado.
   - Generará un archivo `vuelos_history.json` dentro del repositorio del dashboard.

2. **Ventajas**:
   - **Eficiencia**: Carga instantánea mediante JSON pre-procesado.
   - **Seguridad**: Tokens protegidos en los Secrets de la GitHub Action.
   - **Cero Esfuerzo**: El historial se actualiza solo en cada ronda del scraper.
   - **Universal**: Funciona para cualquier usuario sin instalar nada adicional.
